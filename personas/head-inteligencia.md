# Persona: Head de Inteligencia de Mercado & Dados

> Data team lead. The power user of Market Hub — validates data quality, configures data sources, and translates raw market signals into actionable intelligence for the entire company.

## Profile

| Attribute | Detail |
|-----------|--------|
| **Decision scope** | Data quality standards, analysis prioritization, model deployment, data source selection |
| **Data appetite** | Deep: wants to see raw numbers, trends, source provenance, and freshness indicators |
| **Frequency** | Daily — this is their primary workspace |
| **Tools** | Python/R, Power BI, Tableau, SQL, Git, CONAB/IBGE/BACEN/CEPEA portals |

## Primary Modules

| Module | What they look for | How often |
|--------|-------------------|-----------|
| **Market Pulse** | Data freshness: are BCB feeds updating? Price accuracy validation against CEPEA portal | Daily |
| **Agro News** | Category distribution: are we getting balanced coverage? Any sources going stale? | Daily |
| **Recuperacao Judicial** | Filing volume trends: are dual filters catching the right cases? False positive check | 2-3x/week |
| **Retailers Directory** | Data completeness: coverage across 27 states, missing locations, stale records | Weekly |
| **Dashboard** | Module health: all 11 active, no errors | Daily |

## Routine: Daily Data Quality Check (8:30 AM, after cron runs at 08:00 UTC)

| Step | Task | Automated / Manual |
|------|------|--------------------|
| 1 | Verify cron ran successfully (check Vercel dashboard or sync-all response) | **Automated** — cron runs at 08:00 UTC |
| 2 | Spot-check Market Pulse: is `last_update` today? Are prices within expected ranges? | **Manual** — validation requires domain knowledge |
| 3 | Review Agro News: scan today's new articles, check category classification accuracy | **AI-Assisted** — auto-categorized, human reviews edge cases |
| 4 | Check Recuperacao Judicial for false positives (articles that aren't truly agro-related) | **Manual** — legal context requires human judgment |
| 5 | Log any data quality issues and file fixes | **Manual** |

## Routine: Weekly Source Audit (Friday PM)

| Step | Task | Automated / Manual |
|------|------|--------------------|
| 1 | Open each RSS feed URL in browser to verify it's still active | **Manual** |
| 2 | Compare BCB series data against CEPEA portal for discrepancies | **Manual** |
| 3 | Check `highlighted_producers` table: any producers to add/remove? | **Manual** — business decision |
| 4 | Review archived news in `news_knowledge` table for summary quality | **AI-Assisted** — summaries generated by OpenAI |

## What a Virtual Coworker Can Do

- **Run automated data quality checks** after each cron: flag if any BCB series returned empty, if any RSS feed timed out, if article count dropped below threshold
- **Generate a daily data health report**: sources checked, records updated, errors encountered
- **Propose new RSS feeds** by searching for Brazilian agro news sites and validating their feed URLs
- **Cross-validate commodity prices** against secondary sources (CEPEA website, CONAB bulletins)
- **Auto-detect stale data**: alert if any commodity price hasn't updated in 3+ business days

## What Requires Human Judgment

- Deciding which new data sources to add (editorial quality, relevance)
- Adjusting category classification keywords when they produce false positives
- Validating that judicial recovery cases are truly agro-related
- Choosing which highlighted producers to track (business strategy)
- Interpreting data anomalies that could be real market events vs data errors
